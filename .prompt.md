# Dropdown Menu Style Optimization Plan

## Goal
Unify and optimize the dropdown menu styles in `MainHeader.vue` and `ChatHeader.vue` by extracting the premium "glassmorphism" styles from `MainHeader` into a global CSS file.

## Analysis
- **MainHeader**: Contains premium styles for `.ez-header-popper` and `.ez-dropdown-menu` (glassmorphism/blur/borders) in a non-scoped style block.
- **ChatHeader**: Uses `el-dropdown` but lacks the `popper-class="ez-header-popper"` and the corresponding styles, leading to a default or inconsistent look.
- **Strategy**: Move the shared styles to `src/assets/styles/ez-dropdown.css` and ensure both components use them.

## Changes

### 1. Create `src/assets/styles/ez-dropdown.css`
- Extract the following classes from `MainHeader.vue`:
    - `.el-popper.ez-header-popper` (and children)
    - `.ez-dropdown-menu` (and children items)
- Ensure styles cover both light and dark modes (MainHeader's styles seem to rely on global vars which handle dark mode, but I will verify if specific dark mode overrides are needed).

### 2. Update `src/main.ts`
- Import `import '@/assets/styles/ez-dropdown.css'` so it's globally available.

### 3. Update `frontend/vue-ezchat/src/views/layout/sections/MainHeader.vue`
- Remove the `<style>` block (non-scoped) containing the dropdown styles.
- Keep `popper-class="ez-header-popper"` on the dropdowns.

### 4. Update `frontend/vue-ezchat/src/views/chat/sections/ChatHeader.vue`
- Add `popper-class="ez-header-popper"` to the `<el-dropdown>` component.
- Ensure the `<el-dropdown-menu>` has `class="ez-dropdown-menu"`.

## Verification
- **Manual Test**:
    1. Open the app.
    2. Click the Language/Settings dropdown in the Main Header (Left/Top). Verify styles are preserved.
    3. Go to a Chat Room. Click the "More" (three dots) menu in the Chat Header.
    4. **Check**: The dropdown menu should now have the same glassmorphism effect, rounded corners, and item hover styles as the Main Header.
